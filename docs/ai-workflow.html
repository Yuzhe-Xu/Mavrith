AI workflow guide

Keep model behavior in Python and give agents a smaller map to read.

Mavrith's manifest layer is derived from the Python model. It helps AI agents inspect topology, parameters, source references, descriptions, and execution summaries without turning YAML into a modeling DSL.

Python Is The Source Of Truth

Agents should write or patch Python blocks and systems first. The graph/detail YAML files are review artifacts, not inputs that drive simulation. validate(), compile(), and run() never write manifest files.

Keep Semantics Near The Code

Use parameters=... for stable, serializable tuning knobs and description=... for human-authored semantics that should survive AI edits. 

class Gain(Block):
    inputs = (PortSpec.input("u", spec=FLOAT),)
    outputs = (PortSpec.output("y", spec=FLOAT),)

    def __init__(self, gain: float) -> None:
        super().__init__(
            direct_feedthrough=True,
            parameters={"gain": gain},
            description="Algebraic gain used by the outer-loop controller.",
        )
        self.gain = gain

Export Graph And Detail Manifests

Install YAML support and call write_manifest_bundle() explicitly after meaningful topology, port, parameter, or description changes. 

from mavrith import SimulationConfig, build_detail_manifest, write_manifest_bundle

bundle = write_manifest_bundle(system, ".mavrith-ai")
runtime_detail = build_detail_manifest(
    system,
    config=SimulationConfig(start=0.0, stop=1.0, dt=0.1),
)
  • graph.yaml stays topology-first and lightweight.
  • detail/index.yaml lists every detail shard.
  • detail/<path>.yaml stores ports, parameters, sources, and fingerprints.
  • A root runtime detail can include execution order, rate groups, and time-grid behavior.

Recommended AI Read Order

  1. Open graph.yaml to understand the topology.
  2. Open detail/index.yaml to choose relevant shards.
  3. Open only the needed detail/<path>.yaml files.
  4. Open Python source from instance_source or implementation_source.
  5. Read runtime detail when execution order or timing behavior matters.

Agent Edit Loop

1. Patch Python

Change custom blocks, systems, ports, descriptions, or parameters in source files.

2. Validate

Run Simulator.validate(...) and repair diagnostics before running a simulation.

3. Refresh manifests

Re-export graph/detail files so review artifacts match the Python source of truth.

Review hint: when implementation fingerprints change but description text does not, Mavrith can mark descriptions as review_recommended.
