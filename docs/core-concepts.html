Core concepts

The model is a graph of explicit blocks with explicit signal contracts.

Mavrith supplies the simulation kernel. Domain behavior belongs in custom Python blocks that declare ports, state, timing, and outputs.

Blocks

Base class Use it when State contract
Block The output is a constant or a pure algebraic transform. No committed runtime state is required.
DiscreteBlock The block has sampled state updated on clock hits. Implement initial_discrete_state() and update_state().
ContinuousBlock The block state follows a differential equation. Implement initial_continuous_state() and derivative().

Ports And SignalSpec

Ports are declared as class attributes with PortSpec. Use SignalSpec when a signal has a known dtype and shape. If both ends declare dtype or shape, they must match exactly. 

FLOAT = SignalSpec(dtype="float", shape=())
VECTOR3 = SignalSpec(dtype="float", shape=(3,))

class Gain(Block):
    inputs = (PortSpec.input("u", spec=FLOAT),)
    outputs = (PortSpec.output("y", spec=FLOAT),)
  • One input port can have only one upstream connection.
  • One output port can fan out to many inputs.
  • Unspecified dtype or shape acts as a wildcard.
  • Mavrith does not add implicit casts, broadcasting, or promotion.

Direct Feedthrough

direct_feedthrough tells the compiler whether the current output depends on current inputs.

Use True

The block reads current inputs in output(), such as adders, gains, subtractors, and routing logic.

Use False

The block output comes from stored state or fixed parameters, such as constants, sampled controllers, and continuous plants.

Feedback rule

Feedback through only direct-feedthrough blocks is an algebraic loop. Put state in the path to make the loop executable.

Systems And Subsystems

A System owns named blocks or subsystems and directional connections written as "component.port". A Subsystem exposes a clean boundary while keeping reusable internal structure close to the code. 

system = System("demo")
system.add_block("source", Constant(1.0))
system.add_block("plant", Integrator(initial=0.0))
system.connect("source.out", "plant.u")

Time And Execution

Simulation is fixed-step and deterministic. Discrete blocks declare sample_time, optional offset, and optional priority. Continuous blocks are integrated through the configured solver between fixed time points.

  1. Evaluate visible outputs from committed state at the current time.
  2. Run discrete rate groups that hit the current time, in priority order.
  3. Re-evaluate downstream direct-feedthrough logic after each commit.
  4. Record the visible snapshot and advance continuous state one step.

Validation

Simulator.validate(system, config) checks model structure without starting the simulation. Reports include stable diagnostic codes, precise locations, short suggestions, execution order, rate groups, cross-rate warnings, and signal metadata.

Use validation as the edit loop. Build or patch the Python model, validate, inspect diagnostics, then run once the report is valid.
